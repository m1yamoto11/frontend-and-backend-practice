# Lighthouse Аудит - Задание 18

## Инструкции по проведению Lighthouse аудита

### 1. Как провести аудит contacts.html

1. **Запустите локальный сервер:**
   ```bash
   # Если у вас установлен Python 3:
   python -m http.server 8000
   
   # Или если у вас установлен Node.js:
   npx serve .
   
   # Или используйте Live Server в VS Code
   ```

2. **Откройте страницу в браузере:**
   - Перейдите к `http://localhost:8000/pages/contacts.html`

3. **Запустите Lighthouse:**
   - Откройте DevTools (F12)
   - Перейдите на вкладку "Lighthouse"
   - Выберите все категории: Performance, Accessibility, Best Practices, SEO
   - Выберите "Desktop" или "Mobile"
   - Нажмите "Generate report"

4. **Сохраните отчет:**
   - После завершения аудита нажмите "Save as HTML" или "Save as JSON"
   - Сохраните файл в папку проекта

### 2. Что проверяет Lighthouse

#### Performance (Производительность):
- **First Contentful Paint (FCP)** - время первой отрисовки контента
- **Largest Contentful Paint (LCP)** - время загрузки крупнейшего элемента
- **Cumulative Layout Shift (CLS)** - стабильность макета
- **Speed Index** - скорость отображения контента
- **Total Blocking Time (TBT)** - время блокировки основного потока

#### Accessibility (Доступность):
- Контрастность цветов
- Альтернативный текст для изображений
- Корректная семантика HTML
- ARIA-атрибуты
- Навигация с клавиатуры
- Порядок фокусировки

#### Best Practices (Лучшие практики):
- HTTPS
- Консольные ошибки
- Устаревшие API
- Уязвимости безопасности
- Размеры изображений

#### SEO:
- Meta-теги
- Структурированные данные
- Индексируемость
- Мобильная адаптивность
- Размеры ссылок

### 5. Возможные проблемы и их решения

#### Performance проблемы:
1. **Медленная загрузка CDN ресурсов**
   - Решение: Использовать preconnect, dns-prefetch
   - Решение: Загружать критические CSS inline

2. **Большие изображения**
   - Решение: Сжатие изображений
   - Решение: Использование современных форматов (WebP, AVIF)
   - Решение: Lazy loading

3. **Неиспользуемый CSS/JS**
   - Решение: Удаление неиспользуемого кода
   - Решение: Минификация файлов

#### Accessibility проблемы:
1. **Низкий контраст текста**
   - Решение: Увеличить контрастность до минимум 4.5:1

2. **Отсутствие alt у изображений**
   - Решение: Добавить описательный alt текст

3. **Проблемы с фокусировкой**
   - Решение: Добавить :focus стили
   - Решение: Корректный порядок tabindex

#### SEO проблемы:
1. **Отсутствие meta description**
   - Решение: Добавить уникальное описание страницы

2. **Неоптимизированные заголовки**
   - Решение: Использовать H1-H6 в правильной иерархии

3. **Отсутствие структурированных данных**
   - Решение: Добавить JSON-LD разметку
